## WHAT WE ARE BUILDING

**ReviewIQ** is an AI-powered Customer Review Intelligence Platform. It ingests raw, noisy,
multilingual customer reviews from Indian e-commerce (Amazon/Flipkart), extracts feature-level
sentiment using Claude AI, detects emerging complaint trends using statistical analysis,
scores issues by business urgency, and generates team-specific action recommendations.

**The system must do five things end-to-end:**
1. Ingest messy reviews (CSV, JSON, paste) and clean them including Hinglish text
2. Send batches of 5 reviews to Claude API and extract per-feature sentiment with confidence scores
3. Detect statistically significant trends using sliding windows and Z-score anomaly detection
4. Score each issue by a 6-factor escalation formula and rank by business urgency
5. Generate a team-specific action brief (Quality / Product / Support / Marketing / Operations)
   and display everything on a Streamlit dashboard with a "Decision Mode" toggle

**The entire codebase must run with:** `python run_precompute.py` then `streamlit run frontend/dashboard.py`

## SECTION 5 — BACKEND: DEDUPLICATION

### File: `backend/deduplication.py`

**Purpose:** Cluster near-duplicate reviews using TF-IDF cosine similarity.

**Functions to implement:**

```python
def compute_dedup_clusters(reviews: list[dict], threshold: float = 0.85) -> list[dict]:
    """
    Compute TF-IDF vectors for all reviews using clean_text.
    Use sklearn TfidfVectorizer with max_features=5000, ngram_range=(1,2).
    Compute cosine similarity matrix.
    For any pair with cosine_similarity > threshold:
      - Keep the first occurrence as "representative"
      - Mark all others as near_duplicate=True
      - Group them into a cluster_id string like "cluster_001"
    
    Return same review list with these fields added per review:
      near_duplicate: bool
      cluster_id: str or None
      dedup_method: "tfidf_cosine" or None
    """

def compute_trust_score(review: dict, all_reviews: list[dict]) -> tuple[int, list]:
    """
    Compute a trust score from 0 to 100. Lower score = more suspicious.
    
    Start at 100, subtract penalties:
    
    Signal 1: Near-duplicate text (weight 30)
      If review["near_duplicate"] is True: subtract 30 points
    
    Signal 2: Burst submission pattern (weight 25)
      Count how many reviews have the same username within a 1-hour window.
      If >= 4 reviews from same username in same hour: subtract 25 points
    
    Signal 3: Low lexical diversity (weight 20)
      Compute type-token ratio: unique_words / total_words
      If < 0.4 and total_words > 5: subtract 20 points
    
    Signal 4: Rating-sentiment mismatch (weight 15)
      Rating is 5 but clean_text contains: "terrible", "worst", "broken", "bakwas",
        "bekar", "refund", "return", "never buy", "waste"
      OR rating is 1 but clean_text contains: "excellent", "mast", "zabardast",
        "perfect", "amazing", "love it"
      If mismatch detected: subtract 15 points
    
    Signal 5: Abnormally short content (weight 10)
      If len(clean_text.split()) < 4: subtract 10 points
    
    Clamp result to 0–100.
    
    Return (trust_score: int, suspicion_flags: list[str])
    suspicion_flags is a list of which signals triggered, e.g. ["near_duplicate", "burst_pattern"]
    """

def classify_trust(trust_score: int) -> str:

    trust_score > 70  → "include"
```

### File: `backend/trend_detector.py`

**Purpose:** Divide reviews into windows of 50. Compute per-feature complaint/praise rates

**Functions to implement:**

```python
def assign_windows(reviews: list, window_size: int = 50) -> list:
    """
    Sort reviews by timestamp ascending.
    Assign window_label to each review: "W1", "W2", "W3", etc.
    reviews[0:50] → W1, reviews[50:100] → W2, etc.
    Return same list with window_label field added.
    """

def compute_window_rates(reviews: list) -> dict:
    """
    For each (product, feature, window) combination, compute:
      complaint_rate = negative_mentions / total_mentions
      praise_rate = positive_mentions / total_mentions
      mention_count = total mentions of this feature in this window
    
    Input: reviews with features already extracted (from sentiment_engine output stored in DB)
    
    Return nested dict:
    {
      "SmartBottle Pro": {
        "packaging": {
          "W1": {"complaint_rate": 0.06, "praise_rate": 0.80, "mention_count": 12},
          "W2": {"complaint_rate": 0.14, "praise_rate": 0.72, "mention_count": 14},
          "W3": {"complaint_rate": 0.38, "praise_rate": 0.44, "mention_count": 18}
        }
      }
    }
    """

def compute_zscore_anomalies(window_rates: dict, zscore_threshold: float = 2.0) -> list:
    """
    For each (product, feature), compute rolling mean and std of complaint_rate across windows.
    Compute Z-score for the latest window: (latest_rate - mean) / std
    
    If only 2 windows exist, compute simple delta instead of Z-score.
    If only 1 window exists, skip.
    
    Flag as anomaly if Z-score > zscore_threshold.
    Flag as extreme anomaly if Z-score > 3.0.
    
    Return list of alert dicts:
    [
      {
        "product": "SmartBottle Pro",
        "feature": "packaging",
        "alert_type": "emerging_issue",  # see classification below
        "z_score": 3.1,
        "baseline_rate": 0.06,
        "latest_rate": 0.38,
        "delta_pp": 32,            # percentage point increase
        "window_rates": {...},     # full window history
        "severity": "extreme"      # "normal" | "anomaly" | "extreme"
      }
    ]
    """

def classify_alert_type(baseline_rate: float, latest_rate: float,
                         prev_rate: float, z_score: float) -> str:
    """
    Classify the alert into one of 5 types:
    
    "emerging_issue"  → complaint rate doubles OR z_score > 2.5
    "sentiment_drop"  → feature drops from praise > 65% to praise < 50% across 2 windows
    "new_mention"     → feature appears in < 5% baseline but now > 20%
    "praise_spike"    → positive rate jumps > 25 percentage points
    "volatility"      → sentiment oscillates (alternating high/low across 3+ windows)
    
    Return the matching type string. If none match, return "watch".
    """

def compute_trend_deltas(window_rates: dict) -> dict:
    """
    For each (product, feature), compute:
      delta_CB = latest_window_rate - previous_window_rate
      delta_CA = latest_window_rate - baseline_window_rate (W1)
    
    Return same nested structure as window_rates with deltas added per feature.
    """
```

## SECTION 8 — BACKEND: LIFECYCLE TRACKER

### File: `backend/lifecycle_tracker.py`

**Purpose:** Determine what lifecycle stage each detected issue is in
based on its complaint rate history across windows.

```python
LIFECYCLE_STAGES = ["detected", "growing", "systemic", "persistent", "resolving", "resolved"]

def compute_lifecycle_stage(issue_history: list) -> str:
    """
    Input: list of (window_label, complaint_rate) tuples, ordered oldest to newest.
    Example: [("W1", 0.06), ("W2", 0.14), ("W3", 0.38)]
    
    Transition rules (evaluate in this exact order):
    
    1. If len(issue_history) < 2: return "detected"
    
    2. baseline_rate = issue_history[0][1]
       rates = all complaint rates in order
       latest = rates[-1]
       previous = rates[-2]
       above_threshold = [r > (baseline_rate * 2 + 0.05) for r in rates[1:]]
    
    3. RESOLVED: if latest <= baseline_rate + 0.05 → return "resolved"
    
    4. RESOLVING: if latest < previous AND latest > baseline_rate + 0.05 → return "resolving"
    
    5. PERSISTENT: if len(above_threshold) >= 3 AND all(above_threshold[-3:]) → return "persistent"
    
    6. SYSTEMIC: if len(above_threshold) >= 2 AND all(above_threshold[-2:]) → return "systemic"
    
    7. GROWING: if latest > previous → return "growing"
    
    8. Default: return "detected"
    """

def get_lifecycle_weight(stage: str) -> int:
    """
    Used by escalation scorer.
    systemic=100, persistent=90, growing=70, detected=40, resolving=20, resolved=5
    """
    weights = {
        "systemic": 100, "persistent": 90, "growing": 70,
        "detected": 40, "resolving": 20, "resolved": 5
    }
    return weights.get(stage, 40)
```
